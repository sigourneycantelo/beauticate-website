# Updating stories

> How to add or edit an editorial story on Beauticate. Written for Claude and Claude Code to follow when Sig or the team paste a draft. Read CLAUDE.md first for the durable rules. This file is the steps.

## When to use this

When someone gives you a story to publish or update: an article, interview, guide or personal essay. If the piece is built around one brand or product, also read `docs/shoppable-story.md`.

## What I need from you

The person publishing provides:

- The draft text, or a link to the Google Doc.
- The category: Beauty, Style, Wellness, Destinations, Living, Interviews or Vodcast.
- The author name, plus a one-line bio and a photo if they're new to the site.
- The hero image, and any in-body images, with rough captions.
- For commercial pieces: the partner, and any products to feature with prices or Shopify and affiliate links.

If any of these are missing, ask for them. Do not guess.

## Steps

1. Apply house style from CLAUDE.md section 5. Australian spelling, no em dashes, no Oxford commas, never the word "genuinely", prose not bullets, short sentences.
2. Set the headline in sentence case or lowercase, EB Garamond, with one word in italic. Write a one-line standfirst.
3. Choose the template. Standard editorial, or the shoppable story in `docs/shoppable-story.md` if it's built around one brand or product.
4. Structure the body: hero image and title, intro, sections with subheads, images with alt text. Float any pull quote a screen away from the line it came from, never directly under it.
5. Add product cards per CLAUDE.md section 8. No button. Own products link to the Shopify product page, affiliates show "$X at [retailer]" and open the retailer in a new tab. Add the affiliate-links footnote if the piece mixes both.
6. Write the metadata. SEO title around 60 characters, meta description around 155, a lowercase hyphenated slug, the category and tags.
7. Add schema (CLAUDE.md section 13). Article always. FAQPage if there's an FAQ. Product if there are shop modules.
8. Add plain, descriptive alt text to every image.
9. Place the story where stories live in the build, set its category, and save it as a draft.
10. Share a preview link and the checklist below, ticked.

## Rules that always apply

- House style: CLAUDE.md section 5.
- Product cards: CLAUDE.md section 8.
- Single-brand pieces: `docs/shoppable-story.md`.
- Never invent prices, links, quotes, statistics or author bios. Ask.
- Preserve the author's words. Add structure around them, do not rewrite their voice.

## Definition of done

- [ ] House style applied (spelling, no em dashes, no Oxford commas, prose)
- [ ] Headline sentence case with one italic word, standfirst written
- [ ] Hero image set, with alt text
- [ ] Pull quotes not sitting next to their source line
- [ ] Product cards correct, own vs affiliate, footnote if mixed
- [ ] SEO title, meta description, slug, category and tags set
- [ ] Schema added (Article, plus FAQPage or Product if used)
- [ ] Every image has alt text
- [ ] Saved as a draft, preview link shared

## If something's missing

Ask for it in one short list. The usual gaps: final prices, product links, author bio and photo, the hero image, and whether the piece is a paid partnership, which needs a disclosure line up top.

## Example request

What the team types:

> New story for Wellness. Draft is in this Google Doc [link]. Author is Camilla Thompson. Hero is the sauna shot. It features two products from our shop, the buj Uplifter and the Tulita Agati, and one Amazon affiliate, the Theragun. Save as draft.

What you do: follow the steps above, ask only for anything missing, and hand back a preview link with the checklist ticked.
